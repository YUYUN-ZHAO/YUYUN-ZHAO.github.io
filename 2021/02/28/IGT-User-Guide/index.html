IGT User Guide

IGT库在NDI CAPIcommon.lib的基础上进行二次开发。构建igt::NDITrackingDevice类用于与NDI进行通讯，并发送各种指令；构建igt::NavigationData类，用于获取工具的坐标系信息。使用IGT库时需要链接CAPIcommon.lib，并将CAPIcommon.dll放置在目标程序目录下。

IGT代码结构

  • NDI：NDI官方提供的工具类的头文件，在项目中需要include该文件夹；
  • CMakeLists.txt：作为主程序中的一个子模块，编译出IGT.lib供主程序调用，以降低程序耦合性；
  • igtNDITrackingDevice.h/cpp：用于与NDI进行通讯，并发送各种指令（连接、注册、注销、追踪等）；
  • igtNavigationData.h/cpp：用于获取工具的坐标系信息，为每个工具均初始化一个igt::NavigationData类的对象。

IGTDemo项目代码结构

  • build：项目生成路径 ，该路径指定为"where to build the binaries"；

  • lib：项目依赖库，存放CAPIcommon.lib与CAPIcommon.dll；

  • src：项目源码 ；

  • IGT_User_Guide：用户使用手册文档 ；

    注意："where is the source code"指定为当前目录，而非src目录。

工作流程

1. 初始化

初始化一个igt::NDITrackingDevice类的对象，用于连接NDI并完成注册、注销、追踪与获取坐标系数据等任务；为每个工具均初始化一个igt::NavigationData类的对象。

为每个工具m_navigationData绑定追踪设备m_tracker，并载入该工具对应的.rom文件（返回申请到的端口号，若端口号小于0，说明载入失败）

2. 开启追踪

将m_tracker连接NDI设备（判断返回值是否为0，若为0则成功，否则连接失败）。

使用m_tracker开启追踪（判断返回值是否为0，若为0则成功，否则追踪失败）。

3. 获取数据

首先判断当前帧能否获取工具数据（若返回true，则能获取，可调用Get函数获取工具数据，若返回false，则无法获取）。

若能获取，则

  • 调用GetPosition()获取当前帧下工具原点的三维坐标数据；

  • 调用GetOrientation()获取当前帧下工具的坐标系变换矩阵数据（四元数形式）；

  • 调用GetDirection()获取当前帧下工具坐标系指定轴的方向向量：x轴（axis = 0）; y轴（axis = 1）; z轴（axis = 2）。

其中，探针"81(2)"的轴向方向为z轴(axis=2)，探针"NDI"的轴向方向为x轴(axis=0)。
